# 朗读 LRC 歌词组件使用方法

组件名称：**朗读 LRC 歌词**  
组件文件名：`foo_speaklyrics.dll`  
适用软件：foobar2000 1.5/1.6 x86、foobar2000 2.x x86、foobar2000 2.x x64。
主要用途：在播放歌曲时读取 LRC 歌词，并通过屏幕阅读器/Tolk 或 SAPI5.1 朗读当前歌词。

---

## 1. 组件主要功能

本组件可以实现以下功能：

1. 加载本地 `.lrc` 歌词文件。
2. 设置固定的 LRC 歌词目录。
3. 设置临时 LRC 歌词目录。
4. 自动扫描歌词并朗读。
5. 调用外部歌词下载器下载歌词。
6. 支持选择歌词下载来源。
7. 支持搜索 LRC 歌词，并按目录设置保存下载结果。
8. 支持复制歌词，可复制时间戳歌词、无时间戳歌词或分行歌词。
9. 支持按歌词跳转播放位置。
10. 支持添加当前时间歌词，用于手动制作或补全 LRC 歌词。
11. 支持通过 Tolk 调用屏幕阅读器朗读歌词。
12. 支持在没有使用屏幕阅读器时改用 SAPI5.1 语音朗读。
13. 支持按每句歌词长度动态提前朗读。
14. 支持切换同名歌词候选，处理艺术家缺失或版本匹配错误的歌曲。
15. 支持分别配置保益和争渡的主通道、共享通道或自定义独立通道。
16. 支持记录歌词加载、下载、朗读接口和临时文件清理日志。

说明：启动后隐藏 foobar2000 主窗口、减少托盘模式幽灵窗口的问题，已迁移到 `foobar2000 无障碍增强` 组件中设置。

## 2. 安装方法

### 2.1 选择正确的安装包

组件提供两个独立的安装包。选择安装包时只需要确认文件名末尾的架构标记：

- 文件名以 `foo_speaklyrics-版本号-x64.fb2k-component` 结尾：适用于 foobar2000 2.x x64。
- 文件名以 `foo_speaklyrics-版本号-x86.fb2k-component` 结尾：适用于 foobar2000 1.5/1.6 x86 和 foobar2000 2.x x86。

安装包名称中的 `x64` 和 `x86` 必须与 foobar2000 的位数一致。组件安装包分别按架构提供，不使用同时包含两种架构的合并安装包。

### 2.2 使用组件安装包安装

1. 打开 foobar2000。
2. 进入“文件 -> 参数选项 -> 组件”。
3. 选择“安装”，打开与 foobar2000 位数相同的 `.fb2k-component` 文件。
4. 确认安装，应用更改后彻底退出并重新启动 foobar2000。

重启后，可以在组件列表中确认“朗读 LRC 歌词”已经正常加载。

### 2.3 便携版的组件目录

使用安装包安装后，foobar2000 会自动把组件文件放到对应目录，不需要再手动建立架构子目录。

- foobar2000 2.x x64 通常安装到：`foobar2000\user-components-x64\foo_speaklyrics`
- foobar2000 x86 通常安装到：`foobar2000\user-components\foo_speaklyrics`

组件目录中应直接包含 `foo_speaklyrics.dll`、`tolk` 和 `downloader` 等内容，不应在 `foo_speaklyrics` 里面再嵌套一层 `x64` 或 `x86` 文件夹。

### 2.4 首次设置

1. 打开“视图 -> 朗读歌词 -> 朗读歌词设置”。
2. 在“LRC歌词”选项卡的“歌词下载来源”中勾选需要使用的网络来源；如果只使用本地歌词，可以全部取消勾选。
3. 设置“LRC歌词临时目录”。该目录用于保存自动下载的临时歌词和切换同名歌词时取得的候选歌词，默认值为 `%Temp%`。
4. 按需设置“LRC歌词目录”。需要长期保存自动下载的歌词时设置该目录；留空时，歌词只下载到临时目录。
5. 需要自动朗读时，在“常规设置”选项卡中选中“自动朗读”。

自动朗读默认关闭。即使不启用自动朗读，只要勾选了下载来源，组件仍会查找和下载歌词，供复制歌词、按歌词跳转等功能使用。

### 2.5 安装包位数不匹配

如果安装或加载时提示 `Not a valid Win32 application`，通常表示安装包位数与 foobar2000 不一致：

- 64 位 foobar2000 2.x 应安装 x64 包。
- 32 位 foobar2000 1.5、1.6 或 2.x 应安装 x86 包。

改用正确的安装包后，彻底重启 foobar2000。

## 4. 朗读歌词设置

打开：
视图 -> 朗读歌词 -> 朗读歌词设置

设置窗口分为四个选项卡：
1. 常规设置
2. LRC歌词
3. TTS语音
4. 关于

切换选项卡后，焦点会保留在选项卡控件上。按 Tab 可以进入当前选项卡里的控件。

---

### 4.1 常规设置

常规设置里放的是朗读、扫描和切换歌曲播报相关的基础选项。

#### 4.1.1 自动朗读

用于控制是否自动朗读歌词。
- 选中：播放歌曲时自动朗读 LRC 歌词。
- 未选中：不自动朗读，但仍会按设置加载、后台扫描和下载歌词。

默认是关闭。
开启时会播报：
LRC朗读已打开
关闭时会播报：
LRC朗读已关闭

如果开关提示后马上到达新的歌词时间，歌词可以打断“LRC朗读已打开”提示，避免开关提示耽误当前歌词。

说明：自动朗读只控制“是否把歌词读出来”，不控制“是否查找或下载歌词”。  
即使自动朗读关闭，只要“歌词下载来源”里勾选了来源，组件仍会在找不到本地 LRC 时下载歌词。这样可以在不朗读的情况下继续使用复制歌词、按歌词跳转等功能。

如果你不希望组件联网下载歌词，请在“LRC歌词 -> 歌词下载来源”里取消全部来源。

#### 4.1.2 切换歌曲时播报歌曲信息

用于控制切换到新歌曲时，是否先播报歌曲信息。

- 选中：切换歌曲后，通过当前选择的语音输出方式播报歌曲信息。
- 未选中：切换歌曲时不播报歌曲信息，只按原来的歌词朗读逻辑工作。

启用后，组件会优先播报歌曲信息，减少切歌瞬间歌曲信息和开头歌词互相打断的问题。

#### 4.1.3 切换歌曲播报内容

用于选择切换歌曲时播报哪些内容。

可选项包括：
1. 标题和艺术家。
2. 标题。
3. 艺术家和标题。
4. 文件名。
5. 文件名不含扩展名。

如果歌曲缺少标题或艺术家信息，组件会自动回退到文件名或 foobar2000 路径信息。

菜单“查看当前播放歌曲信息”也会使用这里选择的播报内容格式。
这个手动查看功能不受“切换歌曲时播报歌曲信息”复选框影响；即使没有开启自动切歌播报，也可以随时手动播报当前歌曲信息。

#### 4.1.4 切换歌曲播报延迟毫秒

用于控制切换歌曲后延迟多久再播报歌曲信息。

默认值：
0
表示切歌后立即播报。

示例：
300
表示切歌后延迟 0.3 秒播报。

1000
表示切歌后延迟 1 秒播报。

#### 4.1.5 LRC 歌词编码

用于选择 LRC 歌词文件的文本编码。
默认值：
自动

一般情况下建议保持“自动”，组件会自动判断常见歌词编码。
如果某个歌词打开后出现乱码、问号、文字不正常，可以手动选择对应编码后再测试。

目前可选编码包括：
1. 自动
2. Unicode UTF-8
3. Unicode UTF-8 with BOM
4. Unicode UTF-16 LE
5. Unicode UTF-16 BE
6. Simplified Chinese GBK
7. Chinese Simplified GB18030
8. Traditional Chinese Big5
9. ANSI/OEM Japanese Shift-JIS
10. ANSI/OEM Korean
11. Western European Windows-1252
12. Turkish Windows-1254
13. 系统默认 ANSI

说明：
- 带 BOM 的 UTF-8、UTF-16 LE、UTF-16 BE 歌词会优先自动识别。
- 组件下载器下载的歌词通常是 UTF-8，一般不需要手动修改。
- 国内常见旧 LRC 歌词如果乱码，可以优先尝试 GBK 或 GB18030。

#### 4.1.6 歌词朗读方式

可选方式包括：

1. 朗读时间戳歌词：歌词时间戳到达时朗读当前句，这是默认方式。
2. 动态提前朗读歌词：每一句都根据自身文字长度估算朗读时间，在自己的时间戳之前开始朗读。

动态估算会分别考虑中文、日文、韩文字符、英文单词和标点停顿，每句提前时间限制在约 0.9 到 6 秒。为避免相邻歌词顺序颠倒，某句最早不会早于上一句的时间戳。标题、艺术家、作词、作曲等已识别的片头信息不会参与动态提前朗读。

Tolk 无法取得当前屏幕阅读器的真实语速和朗读完成时间，因此目标是大致在每句开始前读完，不能保证毫秒级准确。如果第一句时间戳本身短于估算朗读时间，组件会从歌曲开始时尽快朗读。设定朗读器前置时间仍会在该模式下继续生效，可用于按实际读屏速度微调。

#### 4.1.7 设定朗读器前置时间毫秒

用于控制歌词朗读的时间偏移。
单位：毫秒。

示例：
0
表示不提前也不延后。

-1000
表示提前 1 秒朗读。

1000
表示延后 1 秒朗读。

注意：如果设置为负数，比如 `-1500`，开头时间很靠前的歌词可能会因为提前后变成负时间，从而出现开头歌词不朗读的情况。遇到开头歌词漏读，可以先把这个值改回 `0` 测试。

#### 4.1.8 无lrc歌词后台扫描时间毫秒

用于控制当前没有歌词时，后台多久扫描一次歌词目录或临时目录。

例如：
500
表示每 0.5 秒扫描一次。

3000
表示每 3 秒扫描一次。

如果你的歌词是外部下载器下载到临时目录，建议不要设置太大，否则新歌词下载完成后可能要等比较久才会被发现。

#### 4.1.9 歌词朗读有效时间毫秒

用于控制一行歌词在多长时间内算有效。
默认值：
3000
也就是 3 秒。

这个功能主要用于避免快进、快退时读到已经过期的歌词。
如果设置太大，快进到纯音乐部分时，可能仍然读到前面的歌词。  
如果设置太小，有些歌词可能来不及朗读。

### 4.2 LRC歌词

LRC歌词选项卡里放的是歌词下载来源、下载保存位置、复制歌词方式、歌词文件、歌词目录和临时目录相关设置。

焦点顺序是：
1. 歌词下载来源。
2. 复制歌词默认方式。
3. 设为默认。
4. 歌词分行符号。
5. 复制歌词时过滤片头署名。
6. LRC 文件。
7. 浏览文件。
8. 正式 LRC 歌词目录。
9. 浏览文件夹。
10. LRC 歌词临时目录。
11. 浏览文件夹。
12. 切换歌曲删除临时 LRC 时间毫秒。

#### 4.2.1 歌词下载来源

这里可以勾选自动下载歌词时使用的来源。
如果所有下载来源都没有选中，组件只会走本地歌词扫描，不会调用下载器。

目前可用来源包括：
1. LRCLIB
2. QQ音乐1号源
3. QQ音乐2号源
4. 网易云音乐

首次安装时默认选中 LRCLIB 和 QQ音乐1号源。用户修改来源后，组件会记住新的选择。

实际显示顺序以设置窗口为准。

#### 4.2.2 复制歌词默认方式

用于设置菜单“复制歌词”的默认处理方式。

可选项包括：
1. 按用户选择复制。
2. 复制时间戳歌词。
3. 复制无时间戳歌词。
4. 复制分行歌词。

如果选择“按用户选择复制”，点击“复制歌词”时会让你选择具体复制哪一种格式。
如果选择其他固定方式，点击“复制歌词”时会直接按该方式复制。

#### 4.2.3 设为默认

按钮名称：
设为默认(&F)

快捷键：
Alt+F

这个按钮用于把当前“复制歌词默认方式”和“歌词分行符号”保存为默认复制设置。

#### 4.2.4 歌词分行符号

用于设置“复制分行歌词”时按哪些字符分行。

例如可以填写：
，。！？；,.!?

含义是：复制分行歌词时，遇到这些符号后会换行。

默认留空。
如果留空，复制分行歌词不会额外拆分内容，效果和复制无时间戳歌词一致。

#### 4.2.5 复制歌词时过滤片头署名

默认未选中。选中后，复制歌词时会检查歌词开头的标题、署名和推广信息，例如“歌名 - 歌手”、作词、作曲、编曲、艺术家、音乐总监、制作人、对应英文职务，以及翻唱平台、AI 翻唱、酷狗星曜计划和“一键翻唱”等声明。

字段分隔符支持 `:`、`：`、`∶`、`﹕` 和 `꞉`。组件也会识别连续的“字段：内容”制作名单、较长的中英双语职位，以及使用 `/`、`／` 或 `、` 分隔的多人演唱名单。判断过程会结合字段位置、时间和后续歌词上下文寻找正文起点；确认进入正文后便停止过滤，证据不足时优先保留原文。

该设置适用于时间戳歌词、无时间戳歌词和分行歌词，不会影响朗读、按歌词跳转或原始 LRC 文件。

如果开头没有识别到署名，组件会完整复制歌词，不删除任何内容。

快捷键：
Alt+H

#### 4.2.6 LRC 文件

用于临时手动导入一个 `.lrc` 歌词文件。
这个输入框不会长期记忆手动导入的歌词路径。
选择文件后点确定，会把这个歌词用于当前歌曲。

#### 4.2.7 正式 LRC 歌词目录

这是固定歌词目录。
适合长期保存歌词。
例如：
D:\Lyrics

组件会在这个目录里查找当前歌曲对应的 `.lrc` 文件。
只要这个目录不是空的，自动下载和搜索自动匹配下载的歌词也会保存到这里，并作为长期歌词保留。

如果不想把下载歌词长期保存，请把这个目录留空；组件会自动改用 LRC 歌词临时目录。

#### 4.2.8 LRC 歌词临时目录

这是临时歌词目录。
适合给下载器使用。
默认值：
%Temp%

也就是当前 Windows 用户的临时目录，通常类似：
C:\Users\你的用户名\AppData\Local\Temp

注意：`%Temp%` 会在组件实际使用时自动展开成真实路径，不会当成普通文件夹名称。
你也可以自己改成其他目录，例如：
D:\Temp

组件可以从这个目录读取临时下载的歌词。
切换歌曲时，组件可以按设置删除上一首歌的临时歌词。

自动下载到临时目录的歌词会写入临时清理清单：
foo_speaklyrics_temp_lrc_manifest.txt

该清单保存在 foobar2000 配置目录下。组件启动或退出时，会尝试删除清单里记录的临时 LRC。

注意：
- 组件不会清空整个临时目录。
- 手动放到临时目录里的 `.lrc`，如果没有写入清单，不会被自动清理。
- 如果手动删除清单文件，之前已记录的临时歌词就不会再被组件识别和清理。
- 下次自动下载到临时目录成功后，清单文件会重新创建。
- 不建议把 LRC 歌词临时目录和正式 LRC 歌词目录设置成同一个目录。

#### 4.2.9 切换歌曲删除临时 LRC 时间毫秒

默认值：
0

含义：切换歌曲后立即删除上一首歌的临时歌词。

如果设置为：
1000
表示切换歌曲后延迟 1 秒删除。

这个功能可以避免刚切歌又切回来时重复下载歌词。
注意：停止播放不会删除临时歌词，切换到下一首歌才会删除。

### 4.3 TTS语音

TTS语音选项卡用于选择歌词朗读输出方式。

#### 4.3.1 使用屏幕阅读器

默认选中。

选中时，组件使用 Tolk 调用屏幕阅读器朗读歌词，例如争渡、NVDA 等。  
选中时，不显示 SAPI5.1 的语音选择、语速和试听控件。

如果你主要使用读屏软件，建议保持选中。

选中后会显示以下通道配置控件：

1. 朗读接口：包含“自动”“保益”“争渡”。这个组合框只决定当前编辑哪一种通道配置，不会强制 Tolk 切换读屏；Tolk 仍会自动识别正在运行的读屏。
2. 通道名称：选择保益或争渡后启用。输入“主通道”使用读屏主通道；输入“三方共享通道”使用共享通道；输入其他非空文字则将它作为自定义独立通道名称。默认名称为“朗读歌词通道”。

保益和争渡的通道设置分别保存，切换配置对象不会覆盖另一种读屏。NVDA、JAWS 等没有这里所用的命名通道参数，保持“自动”即可。

更改保益或争渡的通道名称后，点击“应用”或“确定”保存。将名称设为“主通道”“三方共享通道”或其他自定义名称，都会改变对应读屏的通道配置。由于 Tolk 和读屏接口已经加载，新配置需要彻底退出并重新启动 foobar2000 后生效。

#### 4.3.2 保存通道设置并重启

读屏通道设置保存成功后，组件会显示标准确认对话框：

```text
读屏通道设置已保存，重启 foobar2000 后生效。
现在重启吗？

确定(O) 按钮 Alt+O
取消(C) 按钮 Alt+C
```

- 选择“确定(O)”：关闭朗读歌词设置窗口，并让 foobar2000 自动退出后重新启动。
- 选择“取消(C)”：不自动重启，已经保存的通道设置不会丢失。使用新通道设置前，需要由用户稍后彻底退出并重新启动 foobar2000。
- 按 Esc 或 Alt+F4：等同于选择取消，不自动重启。

只有保益或争渡的通道配置发生变化时才会询问是否重启。修改普通歌词设置不会显示这个对话框。如果通道配置写入失败，组件会显示错误提示，也不会执行自动重启。

#### 4.3.3 使用 SAPI5.1 朗读

取消勾选“使用屏幕阅读器”后，会显示 SAPI5.1 相关设置。

可设置内容包括：
1. 选择语音。
2. 语速滑块。
3. 试听文本。
4. 试听(T) 按钮，Alt+T。

选择语音组合框会列出 Windows 中已经安装的 SAPI5.1 语音。
如果列表为空或只有默认语音，说明系统里可用的 SAPI5.1 语音较少。

语速滑块范围是：
-10 到 10

0 是默认语速。  
负数更慢，正数更快。

试听文本会自动生成类似：
你选择了xxxx语音，语速xx。

点击“试听(T)”会使用当前选择的语音和语速播放试听文本。

#### 4.3.4 何时使用 SAPI5.1

如果没有启动屏幕阅读器，或者你想让歌词通过 Windows 语音库朗读，可以取消勾选“使用屏幕阅读器”，改用 SAPI5.1。

如果使用 SAPI5.1 后出现不稳定、无声或语音异常，可以重新勾选“使用屏幕阅读器”，恢复使用 Tolk/读屏朗读。

## 9. 搜索 LRC 歌词

菜单位置：
视图 -> 朗读歌词 -> 搜索LRC歌词
这个功能用于手动搜索并选择歌词版本。已经设置正式 LRC 歌词目录时，下载结果保存到正式目录；正式目录留空时，下载到临时歌词目录。
适合自动下载没有匹配到、或者需要用户自己挑选歌词版本的情况。

### 9.1 窗口控件

搜索窗口包含以下控件：
自动填入当前播放信息(A) 按钮，Alt+A
标题 编辑框
艺术家，可选 编辑框
搜索(S) 按钮，Alt+S
结果 列表
关闭(C) 按钮，Alt+C

### 9.2 搜索方法

1. 打开搜索 LRC 歌词窗口。
2. 如果要使用当前播放歌曲的信息，按“自动填入当前播放信息(A)”按钮。
3. 也可以手动在“标题”和“艺术家，可选”编辑框里输入歌曲信息。
4. 在标题或艺术家编辑框里按回车可以直接搜索。
5. 按“搜索(S)”按钮也可以开始搜索。
6. 搜索时会显示进度条，搜索完成后焦点会进入结果列表。
7. 如果没有搜索到结果，结果列表会显示没有搜索到结果。

如果搜索结果的第一项与填写的标题、艺术家可靠匹配，组件会自动下载该歌词，不需要再从结果列表手动执行下载。没有可靠匹配时，用户可以在结果列表中选择其他版本。

### 9.3 结果列表

结果列表的显示格式类似：
半壶纱，刘珂矣，来源：QQ音乐1号源

用上下光标选择结果。
按回车会下载选中的歌词。正式 LRC 歌词目录不为空时保存到正式目录；正式目录为空时保存到临时目录。
下载完成后不单独提示，因为组件会重新加载当前歌曲并按设置朗读歌词。

### 9.4 下载到固定歌词目录

在结果列表中打开上下文菜单，可以选择：
下载到设置的LRC歌词目录
这个功能会把选中的歌词保存到固定 LRC 歌词目录，适合长期保存。

使用该命令前必须先设置正式 LRC 歌词目录；目录为空时无法执行长期保存。

### 9.5 注意事项

- 搜索使用朗读歌词设置里勾选的下载来源。
- 如果所有下载来源都没有勾选，搜索不会使用网络来源。
- 标题必须填写，艺术家可以不填写。
- 如果没有填写艺术家，组件可以参考当前播放歌曲的艺术家信息辅助匹配。
- 搜索窗口不会重复打开；再次点击菜单会把已经打开的窗口拉回前台。
- 可以用 Esc、Alt+F4 或关闭(C) 按钮关闭窗口。

### 9.6 切换同名歌词

菜单位置：
视图 -> 朗读歌词 -> 切换到上一个同名歌词
视图 -> 朗读歌词 -> 切换到下一个同名歌词

这两个命令适合歌曲只有歌名、缺少或无法正确匹配艺术家时使用。“下一个”会联网加载后一个相同歌名的候选，“上一个”会回到前一个候选；成功后只播报“歌名，艺术家”。候选歌词保存到临时 LRC 歌词目录，不会覆盖正式目录中的歌词。

组件会为当前歌曲缓存并在后台预取同名候选列表，但不会提前下载所有候选歌词。只有执行“上一个”或“下一个”并选中候选后，才会下载对应 LRC。切歌或停止播放后会清理当前候选缓存。

可以在 foobar2000 的“首选项 -> 键盘快捷键”中搜索“切换到上一个同名歌词”和“切换到下一个同名歌词”，分别设置快捷键。已经位于第一个候选时执行“上一个”，会提示没有上一个同名歌词。其他自动下载、复制和朗读设置不受影响。

## 10. 复制歌词

菜单位置：
视图 -> 朗读歌词 -> 复制歌词

这个功能用于复制当前已经加载的 LRC 歌词。

复制方式取决于“朗读歌词设置 -> LRC歌词 -> 复制歌词默认方式”：
1. 按用户选择复制：点击菜单后选择要复制的格式。
2. 复制时间戳歌词：复制带 `[00:12.00]` 这类时间戳的歌词。
3. 复制无时间戳歌词：去掉时间戳，只复制歌词正文。
4. 复制分行歌词：去掉时间戳，并按“歌词分行符号”设置进行分行。

例如原歌词：
[00:12.00]青花瓷
[00:15.00]天青色等烟雨

复制无时间戳歌词后类似：
青花瓷
天青色等烟雨

如果“歌词分行符号”里填写了：
，。

复制分行歌词时，遇到这些符号会换行。

如果“歌词分行符号”留空，复制分行歌词不会额外拆分内容，效果和复制无时间戳歌词一致。

适合复制歌词文本到其他地方使用。

## 12. 添加当前时间歌词

菜单位置：
视图 -> 朗读歌词 -> 添加当前时间歌词

这个功能用于手动制作或补全 LRC 歌词。打开窗口后，组件会把当前播放位置自动填入为 LRC 时间戳。

### 12.1 窗口控件

添加当前时间歌词窗口包含以下控件：
1. 当前时间 编辑框。
2. 刷新当前时间(T) 按钮，Alt+T。
3. 当前时间歌词 编辑框。
4. 编码 组合框。
5. 保存(S) 按钮，Alt+S。
6. 另存为(A) 按钮，Alt+A。
7. 打开(O) 按钮，Alt+O。
8. 预览 编辑框。
9. 关闭(C) 按钮，Alt+C。

窗口不会重复打开；再次点击菜单会把已经打开的窗口拉回前台。

### 12.2 添加歌词方法

1. 播放歌曲到需要添加歌词的位置。
2. 打开“添加当前时间歌词”窗口。
3. “当前时间”编辑框会自动填入类似 `[01:23.45]` 的 LRC 时间。
4. 如果需要重新读取当前播放位置，可以按“刷新当前时间(T)”按钮，或使用 Alt+T。
5. 在“当前时间歌词”编辑框输入歌词正文。
6. 按 Enter，会把“当前时间 + 歌词”追加到“预览”编辑框。
7. 追加后，“当前时间歌词”会清空，“当前时间”会刷新为当前播放时间。
8. 继续输入下一句歌词。

### 12.3 刷新当前时间

“刷新当前时间(T)”按钮用于重新读取当前 foobar2000 播放位置，并写入“当前时间”编辑框。

使用场景：
1. 打开窗口后歌曲已经继续播放，需要重新取当前时间。
2. 手动修改过当前时间后，想恢复为当前播放位置。
3. 暂停后继续播放，需要重新抓取新的时间点。

### 12.4 打开和继续编辑

点击“打开(O)”可以打开已有 `.lrc` 文件。

打开后：
1. 文件内容会显示在“预览”编辑框。
2. 组件会自动识别 LRC 文件编码。
3. 编码组合框会尽量切换到识别出的编码。
4. 后续点击“保存(S)”会直接保存到当前打开的文件。

如果当前预览内容有未保存修改，打开其他文件或关闭窗口前会提示是否保存。

### 12.5 保存和另存为

保存(S)：
- 如果已经打开或保存过文件，会直接保存到当前文件。
- 如果还没有文件路径，会弹出系统保存对话框。

另存为(A)：
- 总是弹出系统保存对话框。
- 保存成功后，当前文件路径会切换到新保存的文件。

保存时会使用“编码”组合框里选择的编码。
新建歌词默认使用 Unicode UTF-8。

### 12.6 预览编辑框

“预览”编辑框是可编辑的多行编辑框。

你可以在这里：
1. 修改错字。
2. 删除某一行歌词。
3. 调整歌词顺序。
4. 补充作词、作曲等普通文本。

示例：
[00:02.67]哨笛：吴题
[00:12.30]风吹过山岗
[00:18.40]旧梦还在远方

### 12.7 注意事项

- 当前不会在保存后自动绑定为当前朗读歌词。
- 如果想让保存后的歌词被当前歌曲自动识别，建议保存到歌曲同目录、设置的 LRC 歌词目录，或按需要手动加载。
- 如果当前没有播放歌曲，当前时间会显示为 `[00:00.00]`。
- 可以用 Esc、Alt+F4 或关闭(C) 按钮关闭窗口。

## 13. 朗读输出方式

组件支持两种朗读输出方式：
1. 使用屏幕阅读器。
2. 使用 SAPI5.1。

默认使用屏幕阅读器。这个方式通过 Tolk 调用屏幕阅读器朗读歌词。
正常安装 `.fb2k-component` 组件包时，所需的 Tolk 和读屏支持 DLL 已经包含在安装包中，不需要用户另外复制。

组件会优先从以下位置加载：
`foo_speaklyrics\tolk\Tolk.dll`

组件启动后会在后台预加载 Tolk 并检测正在运行的屏幕阅读器。首次调用暂时失败时会进行短间隔重试；新的歌词、切歌或播放位置变化会取消已经过期的朗读重试。

当前随组件打包的 Tolk 来源仓库为：https://github.com/boz700908/tolk

如果使用屏幕阅读器时没有声音，请检查：
1. “使用屏幕阅读器”是否已经选中。
2. 屏幕阅读器是否已经启动并可以正常朗读其他程序。
3. 安装包架构是否与 foobar2000 一致。
4. 组件的 `tolk` 子目录和其中的读屏支持 DLL 是否完整。
5. `foo_speaklyrics.log` 中是否有 Tolk 预加载、读屏检测、调用失败或重试记录。

如果不想使用屏幕阅读器，可以在“朗读歌词设置”的“TTS语音”选项卡里取消勾选“使用屏幕阅读器”，然后选择 SAPI5.1 语音和语速。

注意：组件的 SAPI5.1 朗读是独立实现的。当前组件不使用 Tolk 自带的 SAPI5 托底朗读。

## 14. 外部下载器

组件可以调用外部歌词下载器下载歌词。
下载器使用 .NET Framework 4.8 编写。
所以电脑需要安装：
.NET Framework 4.8 运行库
Windows 11 通常已经包含可用的 .NET Framework 4.8；部分 Windows 10 系统可能需要通过 Windows Update 或微软安装程序补充。

下载器已经包含在组件安装包的 `downloader` 子目录中。下载器负责从网络来源搜索歌词：正式 LRC 歌词目录不为空时保存到正式目录，正式目录为空时保存到临时目录。组件随后加载下载器返回的实际 LRC 文件。

## 15. 常见问题

### 15.1 为什么开头歌词少读了？
可能是“设定朗读器前置时间毫秒”设置了负数。
例如：
-1500
如果歌词第一句时间是：
[00:02.67]
提前 1.5 秒后可能太靠前，导致开头歌词处理异常。
解决方法：先改成：
0
再测试。

---

### 15.2 为什么下载歌词后要等一会儿才朗读？

可能是后台扫描时间设置较大。
可以把：
无lrc歌词后台扫描时间毫秒
设置成较小的值，例如：
500
表示 0.5 秒扫描一次。

---

### 15.3 为什么有些 LRC 是这种格式？

例如：
<00:26.223>瓶<00:26.467>身<00:26.736>描<00:27.071>绘
这是逐字时间戳歌词，也叫逐字歌词/卡拉 OK 歌词。
组件主要按普通 LRC 行时间戳朗读，如果来源返回逐字歌词，显示和处理效果可能和普通 LRC 不完全一样。

---

### 15.4 手动加载的歌词会用于其他歌曲吗？

不会。手动加载的 LRC 只绑定到当前播放歌曲，切换歌曲后组件会重新执行歌词查找。

需要长期自动匹配时，应把歌词放到歌曲同目录或正式 LRC 歌词目录。正式目录中的歌词建议使用“歌手 - 歌名.lrc”命名。

如果切换歌曲后仍然加载上一首手动歌词，属于异常情况。请保留当前歌曲信息和 `foo_speaklyrics.log` 以便排查。

---

### 15.5 全部下载来源都不选会怎样？

全部不选时，组件不会调用下载器，只扫描本地歌词目录、临时目录和歌曲同目录歌词。
适合只想用本地歌词的用户。

---

### 15.6 为什么提示“当前没有已加载的 LRC 歌词”？

自动搜索和下载歌词是后台进行的，刚开始播放歌曲时可能需要等待一至数秒。出现该提示时可以：

1. 稍等片刻后再次执行复制歌词或按歌词跳转。
2. 检查“LRC歌词临时目录”和“LRC歌词目录”是否有效。
3. 检查“歌词下载来源”是否至少勾选了一项；如果全部未选中，组件只查找本地歌词。
4. 检查当前歌曲的标题、艺术家或本地文件名是否正确，以免歌词无法匹配。

如果下载已经完成但仍然无法加载，请查看 `foo_speaklyrics.log` 中的查找、下载和解析记录。

## 16. 推荐配置

如果你主要想自动下载并朗读歌词，推荐这样设置：
自动朗读：开启
LRC歌词编码：自动
歌词朗读方式：朗读时间戳歌词，或按需要选择动态提前朗读歌词
设定朗读器前置时间毫秒：0
歌词朗读有效时间毫秒：3000
无lrc歌词后台扫描时间毫秒：500
LRC歌词临时目录：%Temp%
下载来源：首次安装默认选中 LRCLIB 和 QQ音乐1号源，也可以按需要选择其他来源
正式LRC歌词目录：需要长期保存下载歌词就设置；只想临时使用就留空
TTS语音：使用屏幕阅读器保持选中

如果你不想联网下载，只想用本地歌词，推荐这样设置：
自动朗读：开启
LRC歌词编码：自动
歌词朗读方式：朗读时间戳歌词
LRC歌词目录：你的歌词目录
LRC歌词临时目录：可以不设置
下载来源：全部取消勾选
TTS语音：按需要选择屏幕阅读器或 SAPI5.1

## 17. 歌词匹配与文件命名建议

### 17.1 自动匹配规则

组件会结合歌曲标签和本地文件名查找歌词：

1. 当前歌曲同时有标题和艺术家时，优先同时匹配歌名与歌手。
2. 当前歌曲没有有效艺术家时，使用歌名查找歌词。
3. 标题或艺术家是音乐网站的无效占位文字时，会尝试从本地音频文件名取得可用信息。
4. 播放过程中修改歌曲的标题或艺术家标签后，组件会重新匹配当前歌词，不需要通过上一首、下一首来回切换触发刷新。
5. 同名歌曲匹配到错误歌手或版本时，可以使用“切换到上一个同名歌词”和“切换到下一个同名歌词”选择其他候选。

如需使用同名歌词切换，可以在 foobar2000 的“参数选项 -> 键盘快捷键”中搜索这两个命令，并分别设置全局快捷键。

### 17.2 文件命名建议

为了提高本地歌词的匹配成功率，建议使用以下文件名格式：

```text
歌手 - 歌名.lrc
```

例如：

```text
刘珂矣 - 半壶纱.lrc
周杰伦 - 青花瓷.lrc
```

如果歌词与本地歌曲放在同一目录，也可以让歌词文件与音频文件使用相同的主文件名，例如 `青花瓷.mp3` 对应 `青花瓷.lrc`。

文件名中的特殊空格、全角空格或无关符号可能影响匹配。匹配不正确时，应先检查歌曲属性中的 `TITLE` 和 `ARTIST`，再尝试切换同名歌词候选。

## 18. 日志和故障排查

组件会生成日志文件，方便排查无法朗读、无法下载、无法加载歌词等问题。

日志文件位置：
foobar2000 配置目录下的：
foo_speaklyrics.log

临时歌词清单文件位置：
foobar2000 配置目录下的：
foo_speaklyrics_temp_lrc_manifest.txt

如果日志文件超过约 2MB，组件会自动把旧日志轮换为：
foo_speaklyrics.log.old

绿色版 foobar2000 常见位置示例：
foobar2000\profile\foo_speaklyrics.log

说明：foobar2000 2.x 安装版不一定有程序目录下的 `profile` 文件夹。组件会通过 foobar2000 SDK 获取实际配置目录，安装版通常在 `%AppData%` 下，便携版通常在程序目录的 `profile` 下。

如果你需要反馈问题，建议提供：
1. 当前播放歌曲名和艺术家。
2. 当前使用的 LRC 文件或临时目录。
3. `foo_speaklyrics.log`。
4. 如果有崩溃，再提供 foobar2000 的 crash reports。

日志里主要记录：
- LRC 是否找到。
- LRC 是否加载成功。
- LRC 解析失败原因。
- 下载器是否启动成功。
- 下载器路径是否缺失。
- 下载输出目录是否不可用。
- Tolk.dll 是否加载成功。
- Tolk 是否完成预加载以及检测到哪一种屏幕阅读器。
- 自动朗读开关提示或普通歌词任务是否调用失败、重试、被替换或取消。
- 保益、争渡通道配置是否写入成功。
- foobar2000 自动重启命令是否执行成功。
- SAPI5.1 是否朗读失败。
- 临时歌词清单清理是否删除成功。

为避免泄露歌词内容，语音任务日志只记录任务类型和执行状态，不记录正在朗读的歌词正文。

启动后隐藏主窗口相关日志已迁移到 `foobar2000 无障碍增强` 组件。
